  ดึง schema จาก Supabase REST API (/rest/v1/ พร้อม Accept-Profile: workpage)
  พบ 4 ตารางในระบบ :

  [enroll]
    - users_id    : integer (PK, FK → users.users_id)
    - subject_id  : text    (PK, FK → subject.subject_id)
    - enroll_day  : text
    - start_time  : time
    - end_time    : time

  [subject]
    - subject_id     : text (PK)
    - subject_name   : text (required)
    - subject_detail : text

  [work]  ← ตารางหลักสำหรับหน้า detail
    - work_id         : integer (PK, auto-increment)
    - work_title      : text    (required)
    - work_date_start : date    (default: CURRENT_DATE)
    - work_date_end   : date
    - work_detail     : text
    - work_status     : boolean (default: false = ยังไม่เสร็จ)
    - subject_id      : text    (FK → subject.subject_id)

  [users]
    - users_id          : integer (PK)
    - users_name        : text (required)
    - password          : text (required)
    - users_profile_url : text


## ขั้นตอนที่ 2 : สร้าง Back-end API — GET & POST งาน
วันที่ทำ : 2026-07-06
สถานะ : ✅ เสร็จแล้ว
ไฟล์ : app/api/work/route.js

  GET  /api/work?subjectId=[id]
    → ดึงรายการงานทั้งหมดของวิชานั้น เรียงตาม work_date_end

    → Insert งานใหม่เข้า table work
    → คืนค่า row ที่ insert พร้อม work_id ที่ได้รับ

## ปัญหาที่พบ
  เส้นทาง /dashboard/[userId] ส่ง userId เป็นตัวเลขดิบไปใน URL เช่น
  → /dashboard/3
  ทำให้ผู้ใช้สามารถเดา/แก้ไข userId ใน URL เพื่อเข้าถึงข้อมูลของคนอื่นได้

## วิธีการเข้ารหัสที่เลือก : AES-256 (Symmetric Encryption) ผ่าน crypto-js
  เหตุผลที่เลือก :
  - ใช้งานง่ายใน Next.js (ฝั่ง client และ server)
  - เข้ารหัส/ถอดรหัสได้ทั้งสองทาง (สามารถกู้คืน userId จริงมาใช้งาน API ได้)
  - ไม่ต้องเก็บ mapping table เพิ่มเติมใน database
  - ซ่อน userId จริงจากผู้ใช้ใน URL
  - library : crypto-js (npm install crypto-js)

  ทำไมไม่ใช้วิธีอื่น :
  - Hash (bcrypt, SHA) → ถอดรหัสกลับไม่ได้ ไม่เหมาะสำหรับ URL routing
  - JWT → เหมาะกับ token auth ไม่ใช่ URL parameter
  - Base64 → ไม่ใช่การเข้ารหัส แค่ encode ปลอดภัยน้อย

## ขั้นตอนที่ 8 : เพิ่ม Utility สำหรับเข้ารหัส/ถอดรหัส userId
  วันที่ทำ : 2026-07-16
  สถานะ : ✅ เสร็จแล้ว
  ไฟล์ : lib/encryptId.js

  ติดตั้ง library :
    npm install crypto-js

  โค้ด lib/encryptId.js :
  ─────────────────────────────────────────────────────────
  import CryptoJS from 'crypto-js';

  const SECRET_KEY = process.env.NEXT_PUBLIC_ENCRYPT_SECRET;
  // เก็บค่า SECRET_KEY ใน .env.local
  // NEXT_PUBLIC_ENCRYPT_SECRET=your-secret-key-here (ตั้งค่าให้ยาวและซับซ้อน)

  // เข้ารหัส userId → ได้ ciphertext สำหรับใช้ใน URL
  export function encryptId(userId) {
    const encrypted = CryptoJS.AES.encrypt(String(userId), SECRET_KEY).toString();
    // ต้อง encode เพื่อให้ใช้งานใน URL ได้ (แทน +, /, = ที่มีในผลลัพธ์ AES)
    return encodeURIComponent(encrypted);
  }

  // ถอดรหัส ciphertext จาก URL → ได้ userId จริง
  export function decryptId(encryptedId) {
    const decoded = decodeURIComponent(encryptedId);
    const bytes = CryptoJS.AES.decrypt(decoded, SECRET_KEY);
    return bytes.toString(CryptoJS.enc.Utf8); // คืนค่าเป็น string เช่น "3"
  }
  ─────────────────────────────────────────────────────────

  สิ่งสำคัญ :
  - SECRET_KEY ต้องเก็บใน .env.local และ ห้าม commit ขึ้น git
  - ใช้ encodeURIComponent() ครอบผลลัพธ์ AES ก่อนแปะลงใน URL
    เพราะ AES output มีอักขระพิเศษ (+, /, =) ที่ URL ตีความผิดได้
  - NEXT_PUBLIC_ prefix → ทำให้ใช้ได้ทั้งฝั่ง client และ server ใน Next.js

  ก่อนแก้ไข :

    // → URL เป็น /dashboard/3  (เห็น userId จริง)

    import { encryptId } from '@/lib/encryptId';
    const encryptedUserId = encryptId(userId);
    router.push(`/dashboard/${encryptedUserId}`);
    // → URL เป็น /dashboard/U2FsdGVkX1%2B...  (ซ่อน userId จริง)
