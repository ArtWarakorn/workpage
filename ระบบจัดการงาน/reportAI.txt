# Report: API Login & Bug Fixes

## 1. Bugs Found & Fixed in `/app/login/page.js`
- **Bug 1: Empty `handleChange` Function**
  - **Issue:** The `handleChange` function was empty, meaning the state of the username and password fields was not updating when the user typed.
  - **Fix:** Implemented `setForm({ ...form, [e.target.name]: e.target.value });` so the form state accurately tracks input.
- **Bug 2: Invalid Redirection in `handleSignIn`**
  - **Issue:** The login function tried to redirect using `router.push("/dashboard/${id}")`, but `id` was undefined. Furthermore, no actual login validation was happening.
  - **Fix:**
    - Changed `handleSignIn` to an `async` function.
    - Implemented a `fetch` POST request to the newly created `/api/login` endpoint with the username and password.
    - Upon successful login (HTTP 200 OK), redirected the user correctly to `/dashboard/${data.users_id}`.
    - Handled failed logins by setting an `error` state and displaying an error message above the inputs.

## 2. API Creation in `/app/api/login/route.js`
- **Action:** Created a new POST route for login authentication.
- **Details:** 
  - Retrieves the `username` and `password` from the JSON body.
  - Connects to Supabase to query the `workpage` schema and `users` table.
  - Validates if a user matches the `users_name` and `password`.
  - If a match is found, returns the user data as JSON (`HTTP 200`).
  - If no match or an error occurs, returns an "Invalid username or password" message (`HTTP 401`).

## 3. API Creation in `/app/api/users/[id]/route.js`
- **Action:** Created a dynamic GET route for fetching a single user.
- **Details:** 
  - Expects the `[id]` parameter from the URL.
  - Queries Supabase for a single user where `users_id = id`.
  - Used safely inside the dashboard to load user details like the avatar picture and username dynamically.

## 4. API Creation & Dashboard Logic (`/app/api/enroll/route.js`)
- **Action:** Created a new endpoint (`/api/enroll`) strictly for dashboard calendar management, and updated `/app/dashboard/[userId]/page.js`.
- **Details:**
  - **POST `/api/enroll`**: Receives data from the dashboard form. It first **upserts** the subject (รหัสวิชา, ชื่อวิชา, รายละเอียด) in the `subject` table. Then it **inserts** the schedule (วันที่, เลือกเวลาเริ่ม, เลือกเวลาออก) into the `enroll` table tied to the `users_id`.
  - **GET `/api/enroll`**: Retrieves all enrolled subjects for a specific `users_id`.
  - **Dashboard Updates**:
    - Connected all input fields to React `useState`.
    - Implemented a smooth Save/Clear feature.
    - Updated the calendar grid to dynamically render blocks (`#6366f1` background) with the subject code and subject name exactly matching the exact Day and Time slot saved in Supabase.
- **Project Integrity Check:** Successfully rebuilt the project using `npm run build` to verify that no code or routing mechanics were broken during this process!
